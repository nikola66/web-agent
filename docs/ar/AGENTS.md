# إرشادات المستودع لوكلاء الترميز بالذكاء الاصطناعي

قواعد لأي وكيل (Claude Code، Codex، إلخ) يعمل على `web-agent`.

## أسلوب الهندسة

- إصلاحات جراحية؛ لا تُضف أسطرًا دون حاجة.
- عند تغيير أو إزالة ميزة، نظّف الكود القديم في نفس الخطوة.
- الأقل من الكود أفضل.
- اسأل إن لم تتضح النية.
- عدّل الملفات الموجودة قبل إنشاء ملفات جديدة.
- تعليقات للـ «لماذا» غير الواضح فقط، لا لوصف «ماذا يفعل الكود».

## شكل المشروع

سياق البنية: `docs/ar/ARCHITECTURE.md`. استخدم الوثائق الإنجليزية المرجعية ما لم يطلب المستخدم locale.

نقاط الدخول:

- `src/main.tsx` — جذر React
- `src/core/orchestrator.ts` — دورة حياة الوكيل
- `src/agent/adapter.ts` — جسر الواجهة ↔ runtime Node مضمّن
- `src/agent/runtime/turn.ts` — حلقة LLM الرئيسية
- `src/agent/runtime/tools/registry.ts` — أدوات مدمجة وقدرات

شجرة `src/agent/runtime` **مستثناة من `tsc`**. اعتمد على الاختبارات والتحقق وقت التشغيل.

## قبل الإرسال

- `npx tsc -b --noEmit` نظيف
- `npm test` ينجح
- `npm run build` بدون chunks كبيرة جديدة
- تغييرات UI: دخان في `npm run dev`
