# سياسة الأمان

## الإصدارات المدعومة

يُصان الفرع الافتراضي الأحدث وأحدث إصدار مُوسوم.

| الإصدار | مدعوم |
| --- | --- |
| أحدث `main` | نعم |
| أحدث release | نعم |
| إصدارات أقدم | لا |

## الإبلاغ عن ثغرة

لا تفتح issue عامًا للمشاكل الحساسة.

1. تقرير ثغرة خاص على GitHub إن وُجد.
2. وإلا تواصل مع المشرف عبر قناة الاتصال في الملف الشخصي أو الموقع، مع تسمية الرسالة تقرير أمان.

ضمّن: وصفًا موجزًا، الأثر، خطوات إعادة الإنتاج أو PoC، المتصفح/نظام التشغيل/النشر، وهل يؤثر على العرض المستضاف أو التطوير المحلي أو كليهما.

## موقف الأمان

Web Agent مصمم أصلًا للمتصفح ومحليًا أولًا:

- مساحات العمل والجلسات والذاكرة والمهارات والحالة في تخزين المتصفح
- بيانات الاعتماد مُشفّرة محليًا
- عزل بين الـ profiles
- النشر المستضاف يجب أن يكون نقلًا فقط لطلبات upstream وليس backend تخزين

## إخلاء مسؤولية الشبكة

عند عدم توجيه `web_fetch` و`web_search` عبر TinyFish، تمر الطلبات عبر وكيل مؤقت حسب `VITE_WEBAGENT_LAUNCH_MODE` بسبب CORS.

لا يحتفظ الوكيل بقياس عن بُعد أو سجلات.

لخصوصية أعلى، استنسخ المستودع وشغّله محليًا.

## نطاق مفيد

- فشل عزل workspace
- تسريب بين profiles
- كشف بيانات اعتماد
- استمرار غير مقصود على البنية المستضافة
- معالجة غير آمنة للمسارات أو الرفع أو shell
