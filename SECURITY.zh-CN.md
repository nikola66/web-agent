# 安全策略

## 受支持版本

本项目维护默认分支最新代码与最新发布标签。

| 版本 | 支持 |
| --- | --- |
| 最新 `main` | 是 |
| 最新 release | 是 |
| 更早 release | 否 |

## 报告漏洞

请勿为安全敏感问题公开开 issue。

请通过以下私密渠道之一报告：

1. 若仓库已启用，使用 GitHub 私密漏洞报告。
2. 若无法私密报告，通过项目主页或网站上的联系方式联系维护者，并明确标注为安全报告。

请包含：

- 问题简要描述
- 影响范围与受影响模块
- 复现步骤或概念验证
- 浏览器、操作系统与部署方式
- 问题是否影响托管演示、本地开发或两者

收到有效报告后，将尽快确认、修复并协调披露时间。

## 安全立场

Web Agent 设计为浏览器原生、本地优先：

- 工作区、会话、记忆、技能与运行时状态保存在浏览器存储中
- 凭据本地加密后持久化
- 各 profile 相互隔离
- 托管部署应对上游请求保持仅转发，不作为用户数据的持久化后端

## 网络说明

当 `web_fetch` 与 `web_search` 未通过 TinyFish 路由时，请求经 `VITE_WEBAGENT_LAUNCH_MODE` 控制的临时代理发出，以满足浏览器 CORS 限制。

该代理不保留遥测或日志。

若对隐私特别敏感，请克隆仓库并在本地运行。

## 范围说明

以下类型的安全问题尤其欢迎报告：

- 工作区隔离失效
- profile 间数据泄漏
- 凭据暴露
- 托管基础设施上意外持久化提示、文件或记忆
- 不安全的路径处理、上传处理或 shell 执行行为
